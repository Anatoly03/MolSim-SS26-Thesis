\simpleframe\begin{frame}[fragile]{Interface Equivalent in Rust}
    Rust is informally well known for `fighting OOP principles', avoiding inheritance and abstract classes, encouraging \blue{composition}. Instead, Rust offers \blue{traits} for abstractions.

    \vspace{0.5cm}
    \begin{lstlisting}
pub trait Bark {     // equiv: interface Bark {
    fn bark(&self);  //          void bark();
}                    //        }

pub struct Dog;      // equiv: struct Dog { }

impl Bark for Dog {  // Dog implements Bark
    fn bark(&self) {
        println!("Woof!");
    }
}
    \end{lstlisting}
\end{frame}

\simpleframe\begin{frame}[fragile]{Interface Equivalent in Rust}
    A design decision Rust made is to split the scope of struct methods and methods which implement traits. The syntax is \texttt{impl T} for struct methods and \texttt{impl Trait for T} for trait implementations.

    \vspace{0.5cm}
    \begin{lstlisting}
impl Dog {
    fn new() -> Self { // this is a method of Dog
        Dog
    }
}

impl Bark for Dog {
    fn bark(&self) { // this is a Bark implementation for Dog
        println!("Woof!");
    }
}
    \end{lstlisting}
\end{frame}

\simpleframe\begin{frame}[fragile]{Interface Equivalent in Rust}
    \begin{lstlisting}
impl EvilName for Dog {
    fn name(&self) -> &str { "I am dawg!" }
}

impl GoodName for Dog {
    fn name(&self) -> &str { "Cute little fella!" }
}

fn main() {
    let dog = Dog::new();

    // error[E0034]: multiple applicable items in scope
    println!("{}", dog.name());
    // works, disambiguating the trait
    println!("{}", EvilName::name(&dog));
}
    \end{lstlisting}
\end{frame}

\simpleframe\begin{frame}[fragile]{Force Trait}
    It would be a shame to talk about how good traits are without mentioning how they are implemented in the project.

    \vspace{0.5cm}
    \begin{lstlisting}
pub trait Force {
    fn potential(&self, ...) -> f64;

    // syntax of "default implementation" in traits
    fn force(&self, ...) -> Vec3 {
        let potential = self.potential(...);
        let diff = Particle::position_difference(...);
        let distance = diff.length2();

        -diff * (potential / distance)
    }
}
    \end{lstlisting}
\end{frame}

\simpleframe\begin{frame}[fragile]{Force for Lennard Jones}
    \begin{lstlisting}
pub struct LennardJonesForce {
    epsilon: f64,
    sigma: f64,
}

impl Force for LennardJonesForce {
    fn potential(&self, ...) -> f64 {
        let distance = Particle::distance(...);

        let frac = self.sigma / distance;
        let frac6 = frac.powi(6);
        let frac12 = frac6.powi(2);

        4.0 * self.epsilon * (frac12 - frac6)
    }
}
    \end{lstlisting}
\end{frame}

\simpleframe\begin{frame}[fragile]{Rust Enums and Variants}
    % TODO quote: https://ocaml.org/docs/basic-data-types#variants
    % TODO quote: https://en.cppreference.com/cpp/language/enum

    Rust uses OCaml-like enums which are more powerful than C++ enums in that every variant can branch out into a different data type.

    \begin{columns}[T,onlytextwidth]
        \begin{column}{0.48\textwidth}
            \begin{lstlisting}[aboveskip=0pt,belowskip=0pt,breaklines=false]
pub enum Option<T> {
    Some(T),
    None,
}
            \end{lstlisting}
        \end{column}
        \begin{column}{0.48\textwidth}
            \begin{lstlisting}[aboveskip=0pt,belowskip=0pt,breaklines=false]
pub enum Result<T, E> {
    Ok(T),
    Err(E),
}
            \end{lstlisting}
        \end{column}
    \end{columns}
    \vspace{0.5em}

    This feature allows Rust to encode branching of code flow into the data type, setting a design choice for error handling.

    \begin{columns}[T,onlytextwidth]
        \begin{column}{0.48\textwidth}
            \begin{lstlisting}[aboveskip=0pt,belowskip=0pt,breaklines=false]
if let Some(value) = ... {
    // value is of type T
    ...
}
            \end{lstlisting}
        \end{column}
        \begin{column}{0.48\textwidth}
            \begin{lstlisting}[aboveskip=0pt,belowskip=0pt,breaklines=false]
match file.open("hello.txt") {
    Ok(file) => file,
    Err(e) => ...,
}
            \end{lstlisting}
        \end{column}
    \end{columns}
\end{frame}

\simpleframe\begin{frame}{Rust Error Handling}
    \begin{tabular}{ m{0.46\textwidth} | m{0.46\textwidth} } 
        \textbf{Rust} & \textbf{C++} \\
        \hline
        \texttt{Result<T, E>} & \texttt{try} and \texttt{catch} \\
        \texttt{Option<T>} & \texttt{std::optional<T>} \\
        \hline
    \end{tabular}
    \vspace{1em}

    Rust does not have stack-unwinding exceptions which can be `caught' and handled at runtime. There are two types of error handlers in Rust:

    \vspace{1em}
    \begin{tabular}{ m{0.46\textwidth} | m{0.46\textwidth} } 
        \textbf{Recoverable} & \textbf{Unrecoverable} \\
        \hline
        \texttt{Result<T, E>} & \texttt{panic \!("this is very very bad");} \\
        % {\begin{itemize}
        %     \item \texttt{Ok(T)}: the operation succeeded and returned a value of type \texttt{T}.
        %     \item \texttt{Err(E)}: the operation failed and returned an error of type \texttt{E}.
        % \end{itemize}}
        \hline
    \end{tabular}
    \vspace{1em}

    If a program encounters an \blue{unrecoverable error}, you \blue{panic}. This is equivalent to throwing an error which unwinds back to the main function, only calling destructor logic on the way.
\end{frame}
